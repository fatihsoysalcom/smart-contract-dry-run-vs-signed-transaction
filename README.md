# Smart Contract Dry Run vs Signed Transaction

This Python script demonstrates the difference between a 'dry run' simulation of a smart contract transaction and its actual execution. It shows how a dry run can predict outcomes without altering state, while execution commits changes, highlighting why dry runs are not final.

## Language

`python`

## How to Run

Save the code as `dry_run_simulation.py` and run it from your terminal using: `python dry_run_simulation.py`
